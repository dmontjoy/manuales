Manual de usuario de Delfos

Guía práctica para empezar a trabajar en el sistema Delfos del proyecto Línea 2 del Metro de Lima y Callao. El manual explica las tareas principales y usa capturas reales del sistema.

Bienvenida

Delfos centraliza la gestión social del proyecto: stakeholders, interacciones, reclamos, solicitudes, compromisos, reportes y consultas asistidas. El objetivo de este manual es que un usuario nuevo sepa dónde entrar, cómo orientarse y qué acciones puede realizar sin depender de conocimiento técnico.

En Delfos se recomienda trabajar siempre desde el registro correcto del stakeholder. Así se mantiene la historia completa de contactos, reclamos, solicitudes y compromisos asociados a la persona u organización.

No uses botones de guardar, aceptar, eliminar o combinar si solo estás consultando información. Esas acciones pueden modificar datos reales.

Ingresar al sistema

Abre la URL de acceso del proyecto, escribe tu usuario y contraseña, y presiona Iniciar Sesión. Si tu sesión vence, vuelve a iniciar sesión desde la misma pantalla.

Conceptos clave

Stakeholder

Persona, familia, institución, comercio u organización relacionada con el proyecto.

Interacción

Registro de contacto: llamada, visita, reunión, mensaje u otra comunicación.

Reclamo

Queja formal con seguimiento por etapas. Suele tener código como C-XXXX.

Solicitud

Pedido o requerimiento que debe evaluarse y atenderse.

Compromiso

Acuerdo asumido con uno o más entregables para seguimiento.

Ficha

Pantalla que reúne la historia de un stakeholder en pestañas.

Otros términos frecuentes: Relacionista es quien gestiona la relación; Etiqueta clasifica registros; Evaluación resume posición, poder e interés; Entregable es una acción concreta dentro de un compromiso.

Módulo Inicio

¿Para qué sirve? Para revisar rápidamente el estado general del trabajo y las actividades pendientes.

¿Qué puedes hacer aquí? Ver alertas, identificar módulos con actividad pendiente, revisar indicadores y entrar al módulo correspondiente desde el menú.

Módulo Stakeholders

¿Para qué sirve? Para consultar el directorio de grupos de interés del proyecto.

¿Qué puedes hacer aquí? Buscar stakeholders, filtrar por fechas o etiquetas, abrir una ficha, revisar evaluación, exportar información o crear un nuevo registro si tu rol lo permite.

Tareas frecuentes: cómo hacer...

Buscar un stakeholder

  1. Abre Stakeholders desde el menú lateral.
  2. Usa los filtros de fecha, nombre, organización, etiquetas o texto.
  3. Presiona Buscar y revisa la lista de resultados.
  4. Selecciona el nombre del stakeholder para abrir su ficha.

Consultar el historial de un stakeholder

  1. Abre la ficha del stakeholder desde el listado.
  2. Revisa las pestañas Interacción, Reclamos, Compromiso y Solicitudes.
  3. Abre un registro específico si necesitas ver su detalle completo.
  4. No uses Editar o Combinar si solo estás consultando.

Registrar una interacción

  1. Entra a Interacciones.
  2. Presiona el botón de nuevo registro si tu rol lo permite.
  3. Relaciona la interacción con el stakeholder correcto.
  4. Revisa la información antes de guardar para evitar duplicados.

Dar seguimiento a un reclamo

  1. Entra a Reclamos.
  2. Busca por código C-XXXX, stakeholder o texto.
  3. Abre el reclamo y revisa la etapa actual del flujo.
  4. Prioriza los casos con vencimiento próximo o vencido.

Exportar a Excel

  1. Aplica primero los filtros que quieres exportar.
  2. Presiona el botón de descarga.
  3. Elige si deseas exportar datos o evidencias.
  4. Confirma solo si realmente necesitas generar el archivo. La descarga se procesa de forma asíncrona y llega por correo.

Cerrar sesión

  1. Abre el menú de perfil en la esquina superior derecha.
  2. Selecciona Cerrar Sesión.
  3. Espera a volver a la pantalla de login.

Referencia: Roles de usuario

Administrador

Puede usar todos los módulos, reportes y configuración, incluida la gestión de usuarios.

Relacionista

Gestiona módulos operativos y reportes. No accede a configuración administrativa.

Consulta

Revisa información en modo lectura. No debe crear, editar ni cerrar registros.

Referencia: Glosario

Ficha

Pantalla que concentra la historia de un stakeholder.

Etiqueta

Categoría usada para clasificar registros y facilitar búsquedas.

Relacionista

Usuario responsable de gestionar la relación con stakeholders.

Evaluación

Lectura de posición, poder e interés del stakeholder frente al proyecto.

Entregable

Acción concreta que forma parte de un compromiso.

Evidencia

Archivo o sustento asociado a un registro.
